## Why
- 目前首頁與設定頁的資訊卡分布零碎，狀態、操作與進階功能混雜，使用者需要捲動或跳轉多次才能找到相關操作。
- Wi-Fi 與 HomeKit 頁面重複樣板且缺少統一的狀態摘要，不利於在單一頁面內完成檢視與設定。
- 缺乏明確的版面準則導致後續新增功能時難以保持一致，需要定義新的群組化需求。

## What Changes
- 調整 `/` 首頁，將系統/網路/HomeKit 指標整合為共享的「狀態面板」區塊，並把所有設定連結集中在「操作面板」，減少零散卡片。
- 在 `/wifi`、`/homekit` 建立共用的版面骨架：頂部顯示對應狀態（目前連線、配對狀況），下方再放設定表單，確保功能類似的資訊與操作靠在一起。
- 定義新的 Web 介面需求，要求未來頁面遵循「狀態 + 操作」雙欄或卡片化結構，並在 spec 中紀錄。

## Impact
- 使用者在單一區塊即可看到系統與網路指標，操作負擔下降。
- 開發時有明確的介面佈局準則，後續新增頁面可維持一致性。
- 不涉及新硬體或協議支援，僅更新 Web 介面程式與樣式。
